On Developer Happiness

So there's this point that Gary Bernhardt kind of elides in his Birth and Death of JavaScript talk, when he mentions that Brendan Eich understood the need for a scripting language for Navigator and decided to write a spec for it, and he did it in just 9 days; and with very little change (but much extension) that spec is now JavaScript. A comparison is drawn to Clojure, which Rich Hickey thought about and revised constantly for two and a half years before he even let anyone know he was working on it.

It's not exactly controversial to say that Clojure is a better-designed language than JavaScript.[1] It's also not in serious dispute that JavaScript is the most popular programming language in the world. The fact that JavaScript has a monopoly on the runtime for browser interactivity certainly contributes to its ubiquity, but that doesn't wholly explain the adaptation of the language for non-browser environments, and there's obviously a fair number of languages that compile down to JavaScript which have stronger guiding design principles and/or a proper embrace of some academic CS best practices.

It's also worth remembering that, while the language has had a prominent place in every web developer's stack for the last twenty-one years, before the Obama administration writing JavaScript was something you endured.[2]

My pet theory on JavaScript is that, while it's always had ease of learning at it's core, it's prominence coincided with the dramatic improvement in tools and practices that make it accessible. Three things happened to JavaScript around 2008 that caused a shift in popular opinion:

  • First, jQuery made working with the DOM much easier.
  • Second, the Acid3 test (combined with a freshly robust browser market) put a focus on how far from the mark vendors were on standards compliance.
  • Finally, Google Chrome was released with V8 and Firebug's DevTools out of the box.

The net effect was to turn JavaScript into the language that has the fastest time-to-play of any programming language that has ever existed, ever. If you're learning the basics of code for the first time, or coming to JS from elsewhere and need to build a mental map for transpilation, there's no development environment to set up.
